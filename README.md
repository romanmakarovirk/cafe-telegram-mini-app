# Cafe Telegram Mini App

Предфинальная сборка Mini App для кафе:

- `FastAPI` отдает полное меню, создает заказы и хранит их в БД.
- `aiogram` отправляет тестово оплаченные заказы кассиру в Telegram.
- `index.html` работает как Telegram Mini App с категориями меню, корзиной и тестовой оплатой.
- Локально приложение может работать на SQLite, а в Railway переключается на Postgres через `DATABASE_URL`.

## Что реализовано

- Полное меню перенесено с ваших фото в категории:
  - шашлык
  - первые блюда
  - вторые блюда
  - самса
  - салаты
  - напитки
  - кофе и чай
- Реальные фото пока заменены локальными SVG-заглушками.
- `GET /api/menu` возвращает меню, сгруппированное по категориям.
- `POST /api/create_order` создает заказ в БД.
- `POST /api/orders/{order_id}/confirm-payment` имитирует тестовую СБП-оплату.
- `GET /api/orders/{order_id}` возвращает статус заказа и состав.
- Бот:
  - открывает Mini App через `/start`
  - настраивает `Menu Button` для открытия приложения
  - позволяет назначить чат кассира командой `/admin`
  - отправляет кассиру кнопки `Готовится` и `Готов`
  - уведомляет клиента о смене статуса заказа

## Тестовый сценарий оплаты

- Пользователь оформляет заказ в Mini App.
- Оплата сейчас тестовая: после оформления заказ автоматически подтверждается как `mock paid`.
- Деньги не списываются.
- Кассир получает заказ как оплаченный тестовый и может вести его дальше через Telegram.

## Локальный запуск

1. Создайте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Создайте `.env` на основе `.env.example`.

Минимальная локальная конфигурация:

```env
BOT_TOKEN=
APP_BASE_URL=http://127.0.0.1:8000
WEBAPP_URL=http://127.0.0.1:8000
DATABASE_URL=sqlite:///./app.db
ADMIN_CHAT_ID=
```

3. Запустите приложение:

```bash
export $(grep -v '^#' .env | xargs)
uvicorn main:app --reload
```

4. Проверьте:

- `http://127.0.0.1:8000`
- `http://127.0.0.1:8000/healthz`

Для теста без Telegram можно открыть:

```text
http://127.0.0.1:8000/?user_id=123456
```

## Railway деплой

Проект готов к деплою через GitHub в Railway.

Что нужно в Railway:

- создать новый проект
- подключить GitHub-репозиторий
- добавить Postgres
- выставить переменные окружения:
  - `BOT_TOKEN`
  - `APP_BASE_URL`
  - `WEBAPP_URL`
  - `DATABASE_URL`
  - `ADMIN_CHAT_ID` опционально

Стартовая команда уже подготовлена:

- через `Dockerfile`
- через `Procfile`

Рекомендуемое значение:

- `APP_BASE_URL=https://<railway-domain>`
- `WEBAPP_URL=https://<railway-domain>`

## Telegram настройка

Перед публичным запуском:

1. Выпустите новый токен бота в `BotFather`.
2. Подставьте его в `BOT_TOKEN`.
3. После первого запуска отправьте боту `/admin` из аккаунта кассира.
4. Откройте `/start` и проверьте кнопку Mini App.

## Основные файлы

- `main.py` — API, БД, бот, сидинг меню
- `index.html` — интерфейс Mini App
- `requirements.txt` — Python зависимости
- `Dockerfile` — контейнер для Railway
- `.env.example` — пример конфигурации
