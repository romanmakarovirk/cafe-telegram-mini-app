### Безопасная работа с API — обязательные проверки
При работе с платёжным кодом (routes.py, payments/, bot_handlers.py, workers.py):
1. **TEST_MODE:** код НЕ хардкодит `test_mode=True/False`, берёт из `config.py`
2. **FOR UPDATE:** внутри `with_for_update()` НЕТ `await`/HTTP/Telegram API. Паттерн: читай+коммить → await снаружи → новая сессия
3. **Фискализация:** операция с деньгами = чек(и) по 54-ФЗ. См. @memory/fiscal_rules.md
4. **payment_status:** допустимые переходы строго определены. См. @memory/fiscal_rules.md
5. **Идемпотентность:** retry НЕ создаёт дубликаты (чеков, платежей, уведомлений)

## Опасные паттерны — НЕ ПОВТОРЯТЬ
1. **async I/O внутри FOR UPDATE** — блокирует БД. Паттерн: `сессия+commit → await → новая сессия`
2. **Один чек вместо двух** — возврат по 54-ФЗ = ДВА чека: sell (prepayment) + sell_refund
3. **Нестабильный external_id** — АТОЛ external_id должен быть `order-{id}-prepay`, без UUID
4. **amount=0 vs None** — проверять `is not None`, не `if amount:` (пропустит 0)
